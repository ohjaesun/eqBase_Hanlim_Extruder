# 임플란트 성형기 SW 요구사항 체크리스트

본 문서는 '임플란트 성형기 사용자 요구 규격서(URS)'에서 소프트웨어(SW)와 related된 기능, 제어, 데이터, 보안 요구사항만을 추출하여 정리한 체크리스트입니다.

## 5. 시스템 요구사항 (System Requirements)

### 5.4 시스템 요구사항

- [x] **5.4.1 메시지 확인**: HMI는 기기운전자에게 메시지를 Interfacing을 통하여 정보를 안내해야 함.
- [x] **5.4.1 MES 연동**: 필요 시 MES(Manufacturing Execution System)에 연결 가능해야 하며, MES Terminal에서 시작 및 완료 상태 확인 가능.
- [ ] **5.4.2 Batch Report Data**: 생산된 제품의 이름과 Batch Number 포함.
- [ ] **5.4.2 Batch Report Data**: Batch와 Batch 사이의 날짜 및 시간 기록.
- [ ] **5.4.2 Batch Report Data**: 장비 및 공정 작업조건(Machine and process operating conditions) 기록.
- [ ] **5.4.2 Batch Report Data**: 공정 중 파라미터 변경 시 항목, 변경 전/후 값, 변경 시간 기록.
- [x] **5.4.3 공정 연관 Alarm**: Alarm Schedule에 세부적으로 표시.
- [x] **5.4.4 예외 처리**: 기기 고장 & 오류 시 생산기계운전 즉시 중단.
- [x] **5.4.5 통신 방식**: RS-485 Modbus, TCP/IP, OPC Server/Client 중 선택 (MES 연계 고려).

## 8. 기능 요구사항 (Functional Requirements)

- [ ] **8.11 Report 작성**: 원료 투입량, 완제품 생산량(정품/불량품), 압출된 임플란트 diameter 정보 Report 확인.
- [ ] **8.12 Audit Trail**: Log in/out, Run/Stop, Recipe open/edit 등 설비 운영/설정에 관한 이력 확인.
- [x] **8.13 Trend Graph**: 속도, 온도, Torque, 임플란트 직경, 자동 선별기능 On/Off 등 공정 파라미터 실시간 트렌드 확인.

## 9. 일반 및 디자인 요구사항

### 9.4 제어 및 모니터링 장치

- [x] **9.4.2 제어 시스템**: 장비 On/off 및 운전 항목 조절.
- [ ] **9.4.2 경고 레벨**: Action, Alert Limit 경고 레벨 설정 가능.
- [x] **9.4.2 에러 복구**: Emergency stop 후 복구는 운전자의 Error 메시지 확인 후 복구하는 구조.
- [x] **9.4.3 HMI 조작**: 자동/연동 조작 및 수동 선택 조작 가능.
- [ ] **9.4.4 바이브레이터 제어**: 구조적 장치(바이브레이터) HMI에서 On/Off 가능.
- [x] **9.4.5 설정값 확인**: 온도 및 속도 조절 유닛은 HMI에서 설정값, 현재값 확인 및 기록.

### 9.5 위험상황대비 요구사항

- [x] **9.5.4 비상정지 기록**: 비상 정지 세부내용 HMI Display 및 저장.
- [ ] **9.5.4 비상정지 기록**: 비상 정지 정보 Audit trail에 확인 및 저장.
- [x] **9.5.4 재작동**: 재 작동 시 비상정지 전 상태에서 작동 (작동 전 Error 메시지 유지).
- [x] **9.5.5 알람 리스트**: Air 공급 이상, 온도센서 이탈, 파라미터 이탈, 파츠 조립 불량 등 알람 발생 및 정지.

### 9.6 전기적 측면 요구사항

- [ ] **9.6.9 전원 복구 알람**: 정전 복구 시 Alarm 경고, 시스템 자동 중단.
- [x] **9.6.9 에러 저장**: 제어시스템에 Error 내용 표시 및 저장.

## 9.9 데이터 및 보안 요구사항

### 9.9.1 Interface 요구사항

- [ ] **9.9.1.5 네트워크**: SCADA Network 대역대 접속 가능 (192.168.100.XXX).
- [x] **9.9.1.6 데이터 연동**: 설비 생성 Data가 타 시스템에 Interface 되도록 프로그래밍.
- [x] **9.9.1.11 통신 프로토콜**: Open Modbus TCP (우선), Modbus RTU, OPC UA 등 지원.
- [x] **9.9.1.12 Modbus 데이터**: Real 값은 INT로 변환하여 제공 (Modbus 사용 시).
- [ ] **9.9.1.16 시간 동기화**: NTP Server 또는 SCADA 송신 시간으로 기준시간 설정. (연결 끊김 시 자체 시간 -> 복구 시 자동 동기화)
- [ ] **9.9.1.17 가동 상태 신호**: 전원 연결 시 SCADA system에서 "on" 상태 확인 신호 송출.
- [x] **9.9.1.19 Runtime 연동**: Windows용 제어 어플리케이션(Runtime) 정보 Interface.

### 9.9.2 Batch Control 요구사항

- [ ] **9.9.2.1 Batch Report 출력**: Print out 및 PDF export 기능.
- [ ] **9.9.2.2 데이터 저장 기간**: HMI Flash Memory (1년 이상) 또는 Windows 저장장치 (무제한/기간 명시).
- [x] **9.9.2.3 데이터 이동**: USB로 데이터 이동 가능 (가독성 확보).
- [ ] **9.9.2.4 데이터 재현성**: Export된 Data는 Import하여 확인 가능하거나 전용 Browser로 확인.
- [ ] **9.9.2.5 데이터 보호**: 전원 Shutdown 시 작업 중 Batch Data 초기화 방지.
- [ ] **9.9.2.6 삭제 방지**: Batch Report 삭제 불가능 (또는 권한 관리/Audit trail 기록).
- [ ] **9.9.2.7 Batch 기간 설정**: Start 시점과 End 시점 지정 가능.
- [ ] **9.9.2.8 Report 항목**: Recipe(Ver, Param, SV), PV, SV 변경, Batch no, Machine no, User(ID/Level), 가동시간(Start/End, KST), Count(양품/불량 등).
- [ ] **9.9.2.9 Recorder**: Batch Report 미지원 시 대체 Recorder 제공 (Digital, HMI 시간 동기화, CFR Part 11 준수, 저장공간 확보, I/O 채널 여유, PDF/Print 지원).

### 9.9.3 Audit Trail 요구사항

- [ ] **9.9.3.1 Audit Trail 출력**: Print out 및 PDF export.
- [ ] **9.9.3.2 저장 기간**: HMI (1년 이상) 또는 Windows (무제한/기간 명시).
- [x] **9.9.3.3 데이터 이동**: USB 이동 가능 (가독성 확보).
- [ ] **9.9.3.4 데이터 재현성**: Import 확인 또는 전용 Browser 확인.
- [ ] **9.9.3.5 데이터 보호**: 전원 Shutdown 시 마지막 기록 초기화 방지.
- [ ] **9.9.3.6 자동 생성/삭제 불가**: Audit-trail 자동 생성, 삭제 불가능.
- [ ] **9.9.3.7 포함 항목**: Event 발생 시간(KST), 유형, User, 설명, Login/out, Param/Recipe 변경 이력 등.

### 9.9.4 시스템 보안 및 자동제어

- [x] **9.9.4.1 사용자 권한**: 계정 등록 및 승인된 사용자만 제어 권한 부여.
- [x] **9.9.4.2 로그인 잠금**: 로그인 실패 횟수에 따른 잠금 기능.
- [x] **9.9.4.3 잠금 해제**: System owner(Admin)만 해제 가능.
- [x] **9.9.4.4 비밀번호 변경**: 잠금 해제 후 사용자가 직접 비밀번호 변경.
- [ ] **9.9.4.5 자동 잠금**: 대기 시간 초과 시 자동 잠금.
- [x] **9.9.4.6 계정 생성 제한 없음**: (제한 시 최대 수 기재).
- [x] **9.9.4.7 계정 정보**: ID, 성명, 권한 등급/그룹.
- [x] **9.9.4.8 권한 관리**: 계정 생성 시 권한/그룹 선택 (Admin, Supervisor, Operator 등).
- [ ] **9.9.4.9 사용자 목록**: 시스템 내부에서 User List 확인 가능.
- [x] **9.9.4.10 비밀번호 규칙**: 변경 주기, 연속성, 자리 수, 문자열 방지, 영문+숫자 등 설정.
- [ ] **9.9.4.11 전자 서명**: 데이터 승인을 위한 전자 서명 사용.
- [ ] **9.9.4.12 전자 서명 정보**: ID, 이름, 일시(KST), 목적(Meta data).
- [ ] **9.9.4.13 수정 불가**: 전자서명 후 수정 불가능.
- [ ] **9.9.4.14 Rewrite 방지**: Capture된 Data, Report 등은 Rewrite 방지 설계.
- [ ] **9.9.4.15 시간 오차**: 3분/1년 이하 (전원 Off/네트워크 단절 시 포함).
- [x] **9.9.4.16 Windows 기반 제어 시**: 자동 로그인->HMI 전환, 작업표시줄 불가, Runtime 종료 권한(Admin), HMI-Windows 시간 동기화.
- [ ] **9.9.4.17/18 데이터 보존 구조**: 생성/보존 역할 분리 또는 단일화 시 OS Multi User 권한 제한(파일 접근 제한).
- [x] **9.9.4.19 Recipe 관리**: 생성/편집/삭제 권한 구분, 확장 가능, 검증(CSV) 수행.
- [ ] **9.9.4.20 파라미터 Tracking**: 주요 Control module 파라미터는 2가지 이상 방법으로 확인 가능하도록 설계 (주 데이터 손실 대비).
- [ ] **9.9.4.21 백업/복원**: OS, App, License, FW/PLC, Config, Batch, Recipe, Audit-trail, Account 등 백업 및 복원 Copy 제공.
- [ ] **9.9.4.22 네트워크 보안**: 사내 망 이용 시 독립 망, 방화벽, 백신 등 충족.
- [ ] **9.9.4.23 부가 장비 제어**: Main HMI 통한 Access Control 또는 개별 FDA 21 CFR Part 11 준수.
